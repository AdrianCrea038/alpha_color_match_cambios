          MANUAL DE FUNCIONAMIENTO - ALPHA COLOR MATCH 2 (V4.2)
================================================================================

Este documento describe la arquitectura, lógica y flujo de datos del sistema
Alpha Color Match 2 para referencia técnica y futuras modificaciones.

1. PROPÓSITO DEL SISTEMA
------------------------
Alpha Color Match 2 es una suite profesional para la gestión de desarrollo de
colores, validación de archivos .txt maestros (NK) y monitoreo de producción
en planta a través de un Dashboard Ejecutivo de alto rendimiento.

2. ARQUITECTURA TÉCNICA
-----------------------
- Frontend: HTML5, CSS3 (Efectos Neón/RGB) y JavaScript Vanilla (ES Modules).
- Backend/Base de Datos: Supabase (PostgreSQL).
- Persistencia: Sistema híbrido Supabase + LocalStorage (caché).
- Estilos: Sistema modular en css/components/ y css/views/.

3. MOTOR GLOBAL DE VALIDACIÓN (CRÍTICO)
---------------------------------------
Archivo: js/modules/nameValidator.js
- window.isValidColorName(nombre): Valida contra catálogo oficial en Supabase.
- window.isValidNK(nk): Valida contra tabla 'master_nks' en Supabase.
- Detección de Duplicados (js/modules/duplicateHandler.js): Filtra registros por 
  'baseName' normalizado. Impide que nombres idénticos con diferentes NKs o 
  valores CMYK entren al sistema sin ser resueltos manualmente.

4. MÓDULOS PRINCIPALES (DETALLE DE ÚLTIMAS MEJORAS V4.2)
---------------------------------------------------

   A. COMPARADOR DE AUDITORÍA PRINCIPAL (js/main.js & js/modules/resultsRenderer.js)
      - Acción Masiva Unificada: Botón "AGREGAR TODO (MASTER + SEC)". Procesa 
        simultáneamente los colores nuevos del Secundario y los faltantes del 
        Master en un solo ciclo de reconciliación.
      - Sistema de Filtros Color-Coded:
        * VERDE: Coincidencias Exactas.
        * AMARILLO: Equivalentes (Nombre igual, CMYK diferente).
        * NARANJA: Solo en el archivo Master.
        * ROJO: Solo en el archivo Secundario (Nuevos).
        * AZUL: Registros marcados para ser AGREGADOS.
        * GRIS: Registros marcados para ser QUITADOS.
      - Interfaz de Acción Persistente: Los botones "Agregar" y "Quitar" 
        permanecen siempre visibles. El estado seleccionado se resalta con 
        clases '.active-green' o '.active-red' y texto de confirmación (✓).

   B. AUDITORÍA CÍCLICO 2 (js/views/cyclicAuditView.js)
      - Comparación Estricta: Maestro vs Target (Empareja por Nombre+NK).
      - Validación CMYK > 100: Bloqueo de seguridad si algún valor supera 100.
      - Modal de Edición Dual: Permite editar simultáneamente el Maestro y el 
        Target en dos columnas separadas.
      - Sincronización: Botón para igualar el Target al Maestro instantáneamente.

5. REGLAS DE ORO (ESTRICTAS)
----------------------------
1. CMYK > 100: Es un error CORRUPTO. Bloquea exportación y se marca en ROJO NEÓN.
2. Normalización: Siempre usar .toUpperCase().replace(/[^A-Z0-9]/gi, '') para Mapas.
3. Validación de Duplicados: Antes de comparar, el sistema DEBE obligar a resolver
   nombres duplicados mediante el modal de corrección (processFileWithValidation).
4. Feedback Visual: Las acciones de usuario deben tener respuesta inmediata (neon).
